# CSAPP（深入理解计算机系统）Lab 驱动学习计划（macOS 原生优先）

适用人群：会 C、使用 macOS、目标是“系统性提升”，并希望以 Lab/动手为主。

> 说明（重要）：CSAPP 官方 Lab 很多默认按 Linux/x86-64 工具链设计。在 macOS 上，部分 Lab 可能因二进制格式/脚本/工具差异而不兼容。
>
> 本计划采用“双轨策略”：
> - **能在 macOS 原生顺滑完成**的实验：优先做（并给出验收标准）。
> - **官方 Lab 在 macOS 上成本过高**的：用 **macOS 原生等价 Lab** 替代（学习目标不变，材料/工具替换）。
>
> 每天投入：**90 分钟**（建议每周 5 天，周末用于补漏/复盘）。

---

## 目录建议（落地）

建议在本机建立如下目录结构（你可按习惯调整）：

## 8 周 Lab 驱动计划（每日 90 分钟）

### Week 1：Data Lab（位运算/整数表示/浮点）— 先把“底层数学”打牢

**目标**：
- 不依赖查资料，能写出常见位运算技巧；
- 能解释溢出、符号扩展、截断；
- 对浮点的 sign/exp/fraction 有可操作的直觉。

**Day 1（90m）**：位运算热身 + 现象验证
- 读：CSAPP 第 2 章中“整数表示/位运算/移位/溢出”核心小节
- 做：写 `demos/bit_print.c`（打印 int 的二进制；验证负数右移行为）
- 验收：能解释“算术右移 vs 逻辑右移”在你机器上的表现

**Day 2（90m）**：Data Lab 题目 1/3（基础题）
- 做：集中刷基础 bit-level 题（掩码、取反、异或技巧等）
- 验收：每题写一句“用了什么等价变形/不变量”

**Day 3（90m）**：Data Lab 题目 2/3（中档：比较/范围/溢出）
- 做：如 `isLessOrEqual`、`addOK` 这类题（以你手头题库为准）
- 验收：能用“符号位 + 溢出条件”说明正确性

**Day 4（90m）**：浮点相关题/难题
- 读：浮点表示（抓住：规格化/非规格化、舍入、NaN/Inf）
- 做：浮点相关题（若你做的是官方 Data Lab）
- 验收：能解释 NaN/Inf/denorm 的基本判定逻辑

**Day 5（90m）**：复盘 + 自测
- 做：自己出 5 道“位运算题”并写解法（非常有效）
- 输出：`notes/week1.md`（10 行总结 + 易错点清单）

---

### Week 2：Cache Lab（局部性/性能直觉）— 让“为什么慢”变得可解释

**目标**：
- 能预测某段代码的 cache 行为倾向；
- 能用访问顺序与 blocking 做出可观优化。

**Day 1（90m）**：局部性对比实验（macOS 原生必做）
- 做：矩阵遍历对比（row-major vs col-major），用 `time` 测差
- 验收：能用空间/时间局部性解释差距

**Day 2（90m）**：Cache 基础 + 尝试官方 Cache Lab（如可跑）
- 读：cache line、命中/缺失、替换策略
- 做：尝试跑你手头的 Cache Lab（若不兼容，转 Day 3 等价练习）

**Day 3（90m）**：等价练习：写一个简化 cache 模拟器
- 做：实现 direct-mapped 或 2-way 组相联 cache 模型（任选其一）
- 验收：能输出 hit/miss 统计，并用样例验证

**Day 4（90m）**：矩阵转置与 blocking 优化
- 做：baseline transpose + blocking
- 验收：能看到时间下降（哪怕 20% 也算成功）

**Day 5（90m）**：复盘
- 输出：`notes/week2.md`（性能坑与优化模板）

---

### Week 3：Bomb Lab（反汇编/调试）— macOS 原生“等价 Bomb”

> 官方 Bomb Lab 多依赖 Linux/ELF 工具链；macOS 采用等价路线：
> 目标不变：**会用调试器读汇编、追控制流、理解栈与参数传递**。

**Day 1（90m）**：lldb 基本功
- 做：准备一个含多函数/分支的 C 程序，练：断点、单步、读寄存器、反汇编、读内存
- 验收：能在汇编层追踪一次完整函数调用

**Day 2（90m）**：从 C 到汇编映射（-O0 vs -O2）
- 做：同函数用不同优化级别编译并对照
- 验收：能解释“优化后变量去哪了/为什么断点行为变化”

**Day 3（90m）**：自制 bomb（3 个关卡）并用汇编解题
- 做：写 3 个输入校验关卡；然后只看汇编 + 调试解出答案
- 验收：不看源码也能解出 3 关

**Day 4（90m）**：栈帧专项
- 做：多参数 + 局部数组函数，观察栈布局
- 验收：能画出栈帧大致结构

**Day 5（90m）**：输出 lldb 速查卡
- 输出：常用 15 条命令 + 典型场景

---

### Week 4：Attack Lab（栈溢出/ROP 思想）— macOS 教学版

> 真实 exploit 会受系统防护影响；此周以“教学版”理解原理为主。

**Day 1**：溢出原理 + 写不安全 demo（仅本机学习）
**Day 2**：观察崩溃、回溯、返回地址位置
**Day 3**：在可控编译选项下复现实验（必要时降低部分保护）
**Day 4**：ret2func 思想（跳到已有函数，而非注入代码）
**Day 5**：总结防护：NX/ASLR/canary 在现象层面的作用

---

### Week 5：Shell Lab（进程/信号/作业控制）— 写一个迷你 shell

**目标**：
- fork/exec/等待；
- 信号处理（Ctrl-C / Ctrl-Z）；
- 简化版作业控制。

**Day 1**：最小 shell（read → fork → exec）
**Day 2**：PATH 搜索 + 参数解析
**Day 3**：重定向（先 `>`）
**Day 4**：信号处理：不杀死 shell 自己
**Day 5**：简化 bg/fg（不追求 bash 完整兼容）

---

### Week 6：Malloc Lab（堆/虚拟内存）— 自制 malloc（教学版）

**目标**：把 VM 与堆管理串起来，理解碎片与合并。

**Day 1**：用 `vmmap` 观察进程内存段变化
**Day 2**：bump allocator（线性分配）
**Day 3**：free list（显式空闲链表）
**Day 4**：合并 coalescing + 对齐
**Day 5**：压力测试 + 半页设计文档

---

### Week 7：Proxy Lab（网络/I/O/并发）— 简化 HTTP 代理

**目标**：用 socket 写一个可用代理；理解 I/O、并发与缓存。

**Day 1**：socket 基础：echo server
**Day 2**：HTTP 请求行 + 核心 header 解析（最小集）
**Day 3**：转发链路：client → proxy → server → client
**Day 4**：缓存（简单 LRU，可选）
**Day 5**：并发版（线程池/队列/锁）

---

### Week 8：并发专项收口（竞态/死锁/性能）

**目标**：能系统识别竞态与死锁，并能写出正确的同步方案。

- 竞态：共享计数器、生产者消费者队列
- 死锁：两把锁顺序构造与修复
- 性能：锁粒度、线程池设计权衡

输出：`notes/week8.md`（一页“并发陷阱与模板”）
